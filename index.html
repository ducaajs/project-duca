01 Project Overview

Project Duca is a fully autonomous, locally-hosted AI job search pipeline running on a purpose-built Ubuntu 24.04 workstation. It scrapes job boards, ingests forwarded email alerts, verifies and enriches leads, scores them against the operator's profile via local LLM, builds per-lead application packets, and delivers a structured Morning Report to Discord. All outputs are filed to Google Drive. Zero cloud dependency. Zero paid API usage.

The system was built around two principles: absolute data sovereignty and human-in-the-loop governance. Cloud-based job tools and auto-applier bots expose sensitive career data to third-party servers while triggering ATS blacklists that silently kill applications. Duca keeps all processing on the operator's machine. The pipeline finds the signal. The operator makes every decision. Nothing ships without human review.

02 Pipeline Architecture
STAGE 1    Scout (python-jobspy) — Web scrape from Indeed + LinkedIn
STAGE 1.3 Email ingest (Gmail) — Parse forwarded alerts (6 sender-specific parsers)
STAGE 1.4 Merge + cross-dedup — Combine sources, remove duplicates
STAGE 1.5 Vault dedup (7-day window) — Suppress previously seen leads
STAGE 2    Gatekeeper verify — URL liveness, ATS tracing, salary lookup, ghost detection
STAGE 3    Analyst (Ollama LLM) — Score leads against operator profile
STAGE 4    Drive upload — Per-lead application packets + manual-check roster
STAGE 5    Vault write — SQLite logging for all scored leads

Only Stage 3 uses the LLM. Everything else is deterministic Python. Orchestrated by a 530-line bash script. Duca (the OpenClaw agent) fires the pipeline via cron, parses the output, formats the Morning Report, and delivers it to Discord.

04 Critical Debugging
Silent System Failure — Root Cause Analysis

Problem: OpenClaw's api: "ollama" mode sends system prompts using role: "developer". Ollama silently drops unrecognized roles. Duca's 24,000-character system prompt vanished on every request with no error output.

Fix: Switched to api: "openai-completions" with an explicit compatibility block routing through role: "system". Full system prompt reception confirmed on first attempt after fix.

Community-Contributed Fixes — OpenClaw Issues #46298 & #49572

Problem: sessions_spawn and sessions_yield fail silently in OpenClaw 2026.3.13. No posted fix existed. Cross-agent messaging required two undocumented configuration gates with no mention in official documentation.

Fix: Built a working replacement using sessions_send with a blocking wait pattern. Published the first documented solutions to both open issues. Documented the undiscovered configuration requirements.

Production Hardening — 24 Bugs Fixed in 4 Nights

LLM scoring runtime reduced from 72 minutes to 8–15 minutes via direct API calls, adaptive batching, and split-retry with recursive halving. Context overflow (97KB merged JSON exceeding 32K budget) resolved by restructuring pipeline stdout. Cross-company contamination in SearXNG enrichment fixed with strong-token relevance filtering.
