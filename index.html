01 Project Overview

Project Duca is a fully autonomous, locally-hosted AI executive job search pipeline designed, built, debugged, and operationalized by AJ Sottile. Running on a purpose-built Ubuntu 24.04 workstation, the system deploys a six-agent AI pipeline powered by locally-running large language models via Ollama — with zero cloud dependency and zero paid API usage.

The system autonomously scouts job leads, vets companies, analyzes fit, writes tailored application materials, audits quality, and delivers a structured Morning Report every day at 8:00 AM ET. All outputs are filed to Google Drive. Duca communicates exclusively via Discord.

04 Critical Debugging Achievement
Root Cause Analysis — Silent System Failure

The Problem: When api: "ollama" is set in OpenClaw configuration, the framework sends the system prompt using role: "developer". Ollama silently drops any message with an unrecognized role — no error, no warning, no log entry. Duca's 24,000-character system prompt vanished on every single request. Tool calls worked because they use role: "tool" — which is why file reads succeeded but full agent reasoning never followed.

The Fix: Switching to api: "openai-completions" with baseUrl: "http://127.0.0.1:11434/v1" and adding an explicit compat block — supportsDeveloperRole: false, supportsReasoningEffort: false — to the Duca model entry in both config files. This routes through OpenClaw's compatibility layer which correctly sends the system prompt as role: "system".

Verification: Discord test "Duca, who are you?" returned full identity, mission awareness, 5-agent pipeline knowledge, Drive output structure, blacklist protocol, and approval requirements — complete system prompt reception confirmed on first attempt after fix.
